# Security Policy

## Important Notice

**This project is developed by a part-time developer, not a dedicated security team.** We do our best to maintain security, but vulnerabilities may exist that we have not identified. We rely on the community to help us find and report them responsibly.

**Please do not exploit vulnerabilities.** If you find a security issue, please report it to us so we can fix it. We are committed to working with you to resolve the issue and will credit you for your responsible disclosure.

## Supported Versions

Only the latest active version of the Cyber Security Test Pipeline is supported for security updates:

| Version | Supported |
| ------- | --------- |
| 2.0.x   | Yes       |
| < 2.0.0 | No        |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not disclose it publicly or via GitHub issues.** Instead, follow these steps:

1. Send an email to the security contact at: **f20240294@pilani.bits-pilani.ac.in**
2. Include a detailed description of the vulnerability, reproduction steps (with proof-of-concept code or requests where applicable), and potential impact.
3. We will acknowledge receipt of your report and coordinate a fix/remediation plan.
4. We request that you follow responsible disclosure guidelines and allow us time to release a patch before public disclosure.

## Response Timelines

**Important:** Since this is a part-time project, response times may vary. We will do our best to acknowledge your report as soon as possible, but please be patient. We are not a dedicated security team and may take longer to respond.

| Severity | Initial Response | Fix Target |
|----------|-----------------|------------|
| Critical | Best effort     | Best effort |
| High     | Best effort     | Best effort |
| Medium   | Best effort     | Best effort |
| Low      | Best effort     | Best effort |

## Disclosure Policy

- We will confirm receipt of your report as soon as we are able.
- We will provide a status update when a fix is released.
- We will credit reporters in the release notes unless they prefer anonymity.
- We ask that you do not publicly disclose the vulnerability until we have released a fix.

## Known Limitations

- We are not professional security auditors and may miss vulnerabilities.
- We may not be able to respond immediately to reports.
- We may not be able to fix all reported vulnerabilities in a timely manner.
- We rely on community reports to identify security issues.

## How You Can Help

If you find a security vulnerability, please report it to us. We will work with you to understand and fix the issue. We appreciate your help in making this project more secure.
